Challenge Details

Project Requirements:

 

Find the Path is a simple mini-game containing the following parts:

 

  • A code matrix populated with hexadecimal numbers. The matrix can have arbitrary dimensions. It is not necessarily square.
  • One or more number sequences are to be found inside the matrix. Each found sequence earns you a certain amount of score points.
  • Maximum length of path in the code matrix.

 

An example of the input data is shown in the picture below:

 

Example of a simple puzzle with one sequence

 

Your task is to find the shortest path in the code matrix that contains sequences with the largest total score. A path is basically a list of matrix cells that are connected to each other. Length of a path means the number of cells in the list. The shorter the path – the fewer cells it has. The picture below demonstrates the solved puzzle with the resulting path marked in red.

 

Example of the solved puzzle

 

Here, we have a path 3A-10-9B, which is 3 cells in length. This path contains the given sequence 3A-10-9B that earns you 10 score points. In this simplest example the path actually matches the sequence. More complex puzzles typically have longer paths containing various different sequences at once.

 

The rules for solving the puzzle are explained in the next section.

 

Solving the Puzzle

Let’s look at the example above. Here, we have only one sequence 3A-10-9B. The goal is to pick the proper cells from the code matrix in proper order to piece together the given code sequence. 

 

The trick is that you can’t choose completely arbitrary cells. You must always start with the top row. Here, we have 5 possible codes to choose from: 3A, 3A, 10, 9B and 72. Suppose you have selected 3A in the second column. It perfectly matches the first element of our code sequence.

 

Solving the puzzle - first move

 

Next, you need to choose one cell from the same column as the previous cell you picked. In our case, this is the second column. Let’s pick code 10 in the fourth row matching the second element in our sequence.

 

Solving the puzzle - second move

 

Finally, you need to pick code 9B from the fourth row to complete the sequence.

 

Solving the puzzle - final move

 

And that’s it. Congratulations on the first solved puzzle! Let’s summarize the learned rules:

 

  1.  At any given moment, you can only choose cells in a specific column or row.
  2. On each turn, you need to switch between column and row, and you have to stay on the same line as the previous cell you picked. This is similar to Snake game. You have to move at right angles.
  3. You always start with picking the first cell from the top row, then you always go vertically down in order to pick the second cell, and then you continue switching between column and row as usual.
  4. You can’t choose the same cell twice. The resulting path must not contain duplicated cells.
  5. The number of picked cells must not exceed the maximum allowed length of the path.

 

Multiple Sequences

There can be more than one sequence to complete. The picture below demonstrates the solution for a puzzle with two sequences.

 

Example of a puzzle with two sequences

 

You can complete sequences in any order. The order in which the sequences are listed doesn’t matter.

 

Wasted Moves

Sometimes we are forced to waste moves and choose additional cells that don’t belong to any sequence. Let’s look at the following example.

 

Example of a puzzle with one wasted move 

 

The sequences 10-D4-72-72 and 3A-10-D4 start with codes 10 and 3A respectively, but there are no such cells in the first row at all! To deal with this issue, we have to waste one move and choose any other cell, e.g., 72 in the third column. After that we will be able to continue moving and successfully complete both sequences. The length of the resulting path, including wasted moves, is 6 cells which still fit the given limit.

 

Notice how the two sequences “overlap” in the resulting path. This is perfectly fine. The more overlapping between sequences – the shorter the resulting path.

 

In some cases, one wasted move is not enough and we need to waste more of them. The next picture shows a tricky case with three wasted moves in total!

 

Example of a puzzle with three wasted moves

 

Here, we have to waste one move before the sequence 3A-9B and two moves between sequences 3A-9B and 72-D4. Also notice how red lines between 3A and 9B as well as between 72 and D4 “cross” previously selected cells D4 and 9B respectively. This is perfectly allowed. You can pick different cells from the same row or column multiple times.

 

Let’s summarize the rules shown above:

 

  1. You can waste 1 or 2 moves before the first sequence.
  2. You can also waste 1 or 2 moves between any neighboring sequences.
  3. However, you cannot break sequences and waste moves in the middle of them.

 

Competing Sequences

Sometimes it isn’t possible to complete all sequences at once within the given path length limit. In this case you have to choose which sequences to complete and which sequences to ignore. This is the time when sequence scores come into play. If you have to choose between multiple sequences, always prefer sequences with the largest sum of scores.

 

For instance, on the following picture we can complete either 9B-3A-72-D4 or 72-D4-3A but not both of them due to the maximum path length of 5 cells. We have chosen the sequence 9B-3A-72-D4 since it grants more score points than 72-D4-3A.

 

Example of a puzzle with competing sequences

 

Puzzles Without a Solution

Finally, there might be puzzles that have no solution at all.

 

Example of a puzzle without any solution 

 

The first sequence 3A-D4 can’t be completed since code matrix doesn’t contain code D4 at all. The second sequence 72-10 can’t be completed for another reason: codes 72 and 10 aren’t located in the same row or column anywhere in the matrix, and therefore can’t form a sequence.

 

Technical Details / Requirements

To get started please read the README file at the root of the Git repository. Additional details about building and running the project can be found there.

 

Existing Source Code

Even though this is a new project you will not be starting from scratch. In the repository you will also find some preexisting source code. You must continue building on top of that preexisting source code and you should NOT DELETE any of the existing source code, files and folders NOR move or rename any of the existing files, classes, methods or variables. However, you’re free to add new source files to the project and extend existing classes with your own methods and variables. 

 

The project provides several existing classes that you should use in your work:

 

  • Matrix represents a code matrix with arbitrary dimensions.
  • Sequence stores a required sequence of codes along with their scores.
  • Path represents a path in the code matrix. It consists of multiple Position objects.
  • Position refers to a particular row and column in the code matrix.
  • PathFinder is a puzzle solver class. It has a method named run() that you need to implement.

 

Solver Implementation

The PathFinder.run() method requires you to calculate the shortest path in the code matrix that contains sequences with the largest total score.

 

The dimensions of the code matrix, number of sequences and length of sequences can be arbitrary. Your code shouldn’t impose any artificial limits on them.

 

While developing the algorithm, use the following rules to compare different paths and determine the resulting one:

 

  1. When comparing different paths, always choose one that yields more score than another. For instance, a path with one completed sequence that earns 50 points is better than a path with two completed sequences that earns 30 points in total.
  2. If you have multiple paths with the same score, choose the shortest one, i.e., having fewer cells than another. The geometric distance between neighboring cells doesn’t matter.
  3. Finally, if you have multiple paths with the same score and length, return any one of them.

 

Performance Considerations

There are multiple ways to implement the puzzle solving algorithm. 

 

One of them is to generate all possible paths in the code matrix, loop through them and choose the best one. This is the easiest but also the slowest approach. Solving larger matrices with larger maximum length of path might take dozens of minutes, if not hours. If you correctly implement such an algorithm, the automatic grading system will accept your solution with an average score which would be enough to successfully pass the challenge. 

 

If the simplest approach feels too easy to you and you want to achieve a perfect grade, try to implement a faster algorithm that doesn’t iterate through all possible paths and can solve puzzles with the following parameters in less than 10 seconds:

 

  • Matrix size: 10x10
  • Number of sequences: 5
  • Average length of sequences: 4
  • Maximum path length: 12
  • At least one solution exists 

 

Error Handling

Your code should properly validate input parameters, handle errors and throw exceptions. The project provides the following exception classes:

 

  • InvalidArgumentError. Throw it if some of input parameters are incorrect. You should validate the following conditions:
    1. Code matrix should be non-empty
    2. There should be at least one sequence
    3. All sequences should be non-empty
    4. All sequences should have a score greater than zero
    5. Maximum path length should be greater than zero
  • PathNotFoundError. Throw it if you can’t find any suitable path in the code matrix, i.e., the puzzle doesn’t have a solution.

 

Testing Your Work

In order to help you test your work, the project provides a small command-line utility with a single Application class. It basically runs PathFinder on different puzzles and prints their solutions or errors to the console.

 

Unlike other preexisting classes, you can freely modify the source code of Application class and write your own test cases. Please use it to carefully test your algorithm on various matrices and sequences before submitting the final code.
